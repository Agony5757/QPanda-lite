## 1) MC-Y/Z/S/RX/RZ 等 ≥4 控非 X 门的 QASM 导出（新增）

### 现状

`translate_qasm2_oir.py:get_QASM2_from_opcode` 对 ≥4 控制位的 **X 门**现在会
自动降级为 Toffoli ladder（Fix 2）。但对其他门（Y/Z/S/RX/RZ/U3 等）配合 ≥4
个控制位时，导出层仍会抛 `NotImplementedError`。

### 影响

- 目前没有调用路径会产生 ≥4 控的非 X 门 QASM 导出（grover/AE 的多控 Z 经
  `_apply_mcz` 拆成 H·MCX·H，MCX 走 Toffoli ladder）。
- 若用户直接写 `circuit.add_gate("Z", t, control_qubits=[a,b,c,d])` 再调
  `c.qasm`，仍会报 `NotImplementedError`（此时建议改用 `c.originir`）。

### 建议处理

等有具体需求出现时，在 `translate_qasm2_oir.py` 的分解表里扩充其他门类型；
或统一引导用户用 OriginIR 导出（原生支持任意宽度多控门）。

**优先级低**。

## 2) `shadow_expectation` 跨 Pauli 字符串的 per-snapshot 缓存（新增）

### 现状

Fix 5（numpy 向量化）消除了 Python 循环的 O(2^n) 开销，但对每个 Pauli 字符串
仍单独遍历所有 snapshot。当同时估计含相同非 I 支撑的多个 Pauli（如 VQE
Hamiltonian），相同 snapshot 的 `counts` 数组会被多次读取。

### 建议处理

引入 `(snapshot_idx, non_I_mask)` → `(keys, vals, non_I_parity_array)` 的跨调用
缓存。仅在 VQE-style 多 Pauli 估计场景有实质收益，Fix 5 已够满足当前测试。

**优先级低**，触发条件：多 Pauli 估计出现性能回归。

## 已关闭条目（归档）

### ✅ `CircuitControlContext` / `DagContext` opcode 回填（Issue #1 + 关联）

- 原症状：`with circuit.control(*controls): circuit.z(t)` 产生的 QASM/OriginIR
  中 CONTROL 块静默丢失；受控门退化为无条件版本。`CircuitDagContext`、
  `set_control`/`set_dagger` 存在同类缺陷。
- 修复：`Circuit` 新增 `_active_controls: List[int]`、`_active_dagger: bool`、
  `_control_stack` 三个实时状态字段；`add_gate` 在写 opcode 时将上下文状态并入
  `control_qubits` / `dagger` 字段（union 合并，重复 qubit 抛 `ValueError`）；
  `measure()` 在活跃 control/dagger 上下文内拒绝调用；上下文管理器改为
  push/pop 操作，保留 `circuit_str` 写入不变。
- 回归保护：`TestContextManagers`（`test_circuit_builder_qcircuit.py`）新增
  10 条测试，覆盖单层 / 嵌套 / double-dagger-cancel / 重复 qubit 拒绝 /
  measure-in-control 拒绝 / `set_control` 路径等场景。

### ✅ `_apply_mcx` 仅支持 ≤3 控制位（Issue #2 + 关联）

- 原症状：`_apply_mcx(circuit, controls, target)` 对 `n ≥ 4` 直接抛
  `NotImplementedError`；QASM 导出层 `get_QASM2_from_opcode` 对 ≥4 控 X 门
  隐式返回 `None`，导致 `opcode_to_line_qasm` 崩溃。
  `amplitude_estimation._multi_controlled_x` 有相同上限 bug。
- 修复：`grover_oracle._apply_mcx` 对 n ≥ 4 使用 clean-ancilla Toffoli ladder
  （workspace 从 `max(controls ∪ {target}) + 1` 起自动分配，`record_qubit` 幂等
  重用）。`translate_qasm2_oir.decompose_mcx_qasm_text` 提供导出层 QASM2 等价分解；
  `opcode_to_line_qasm` 透传 `qubit_num` 并处理 `_MCX_DECOMP_` 哨兵。
  `amplitude_estimation._multi_controlled_x` 同步对齐。
- 回归保护：`test_grover_5qubit_amplifies_target`（5 qubit，marked=13，3 次
  Grover 迭代，断言 P(marked) > 0.8）全覆盖 n ≥ 4 控 MCX 路径。

### ✅ `grover_diffusion.ancilla` 死参数弃用（Issue #3）

- 原症状：`ancilla` 参数在重构后不再被函数体使用，成为静默死参数；
  因存在于 upstream/main，不可直接删除。
- 修复：函数体开头加 `warnings.warn(..., DeprecationWarning, stacklevel=2)`；
  docstring `ancilla` 段标注为"unused, deprecated"。
- 回归保护：`TestGroverDiffusionDeprecation` 新增两条测试：传 ancilla 时确认
  触发 `DeprecationWarning`；不传时确认无告警。

### ✅ `shadow_expectation` 热循环 O(2^n) Python 遍历（Issue #5）

- 原症状：`shadow_expectation` 对每个对齐 snapshot 在 Python 中遍历 `counts`
  字典（最多 2^n 条），再内嵌 O(n) Pauli 字符串解析；n=6 时每次调用含数十万次
  Python 级别循环。
- 修复：用 numpy 位掩码向量化计算：`keys & non_i_mask` → `np.bitwise_count`
  （numpy ≥ 2.0）或 Kernighan popcount fallback → `signs` → `np.sum(vals*signs)`，
  整个 snapshot 一次性完成，无 Python 级循环。
- 回归保护：`TestShadowVectorizedLargeN` 三条测试覆盖 n=6 的
  `ZIIIII`/`IIIIII` 估计，兼作性能回归哨兵。

### ✅ Grover 位序潜伏问题（MSB/LSB + aggregation）
